> 仿照 class_Detection（学生课堂行为识别系统）架构，构建基于 YOLOv8 的交通标志识别桌面应用。

---

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 项目名称 | 交通标志识别系统（Traffic Sign Detection System） |
| 核心功能 | 对图片/视频/摄像头中的交通标志进行实时检测与分类 |
| 参考项目 | `class_Detection`（学生课堂行为识别系统） |
| 目标 | 实现一个与参考项目架构一致、功能对应的交通标志识别桌面软件 |

## 三、数据集方案（中国交通标志）

### 3.1 推荐数据集

#### 主数据集：CCTSDB（中国交通标志检测基准数据集）

| 项目 | 说明 |
|------|------|
| 全称 | Chinese Traffic Sign Detection Benchmark |
| 来源 | 中南大学（HUST）研究团队发布 |
| 数据量 | 约 15,000+ 张真实道路场景图片 |
| 标注格式 | Pascal VOC XML（边界框标注） |
| 类别体系 | 三大类：禁令标志（Prohibitory）、警告标志（Warning）、指示标志（Mandatory） |
| 场景覆盖 | 城市道路、高速公路、乡村道路、不同光照/天气条件 |
| 下载方式 | GitHub 搜索 "CCTSDB" 或 Kaggle 搜索 "Chinese Traffic Sign" |

**选择理由**：
- 标志样式完全遵循中国国家标准 GB 5768《道路交通标志和标线》
- 图片来自国内真实道路场景，与实际应用高度匹配
- 标注质量高，边界框精确，可直接用于 YOLOv8 训练
- 学术界广泛使用，有大量参考文献和预训练结果可对比

#### 补充数据集：TT100K（清华-腾讯交通标志数据集）

| 项目 | 说明 |
|------|------|
| 全称 | Tsinghua-Tencent 100K |
| 来源 | 清华大学与腾讯联合发布 |
| 数据量 | 100,000 张街景图片，30,000+ 个标注实例 |
| 类别数 | 200+ 类（涵盖中国所有常见交通标志） |
| 标注格式 | JSON 格式，含边界框和类别信息 |
| 下载地址 | https://cg.cs.tsinghua.edu.cn/traffic-sign/ |

**选择理由**：
- 数据量大，类别覆盖全面，可补充 CCTSDB 中缺少的细分类别
- 来自腾讯街景数据，场景多样性好
- 适合需要高精度、多类别识别的场景

#### 可选补充：Roboflow Universe

- 在 Roboflow 平台搜索 "Chinese traffic sign" 或 "CCTSDB"
- 可找到已转换为 YOLO 格式的预处理数据集
- 支持一键导出为 YOLOv8 格式，包含 `data.yaml`
- 适合快速验证原型

### 3.2 数据集使用策略

**建议采用 CCTSDB 为主 + TT100K 为辅**的策略：

| 阶段 | 数据集 | 目的 |
|------|--------|------|
| 初期开发 | CCTSDB 单独使用 | 快速验证流程，约 15,000 张，训练速度快 |
| 中期优化 | CCTSDB + TT100K 合并 | 扩大数据量至 45,000+ 张，提升泛化能力 |
| 最终部署 | 根据实际需求裁剪类别 | 只保留实际场景中出现的标志类别 |

**数据合并注意事项**：
- 两个数据集的类别体系需要统一映射（TT100K 有 200+ 类，CCTSDB 约 60 类）
- 需要编写类别映射脚本，将 TT100K 的细分类别合并到 CCTSDB 的类别体系中
- 合并后需重新划分 train/val/test，确保两个数据集的图片均匀分布

### 3.4 标签格式

标准 YOLO TXT 格式（与参考项目一致）：

```
<class_id> <x_center> <y_center> <width> <height>
```

所有坐标归一化到 [0, 1]。每张图片对应一个同名 `.txt` 文件。

## 四、模型训练方案

### 4.1 模型选择

与参考项目一致，使用 **YOLOv8** 目标检测模型：

| 模型变体 | 参数量 | mAP (COCO) | 适用场景 | 推荐 |
|----------|--------|------------|----------|------|
| YOLOv8n | 3.2M | 37.3 | 轻量部署、实时性要求高 | 入门调试 |
| **YOLOv8s** | **11.2M** | **44.9** | **平衡精度与速度** | **推荐** |
| YOLOv8m | 25.9M | 50.2 | 更高精度 | 有 GPU 资源时 |
| YOLOv8l | 43.7M | 52.9 | 高精度 | 大规模部署 |
| YOLOv8x | 68.2M | 53.9 | 最高精度 | 追求极致精度 |

**建议**：开发阶段使用 **YOLOv8s**，兼顾训练速度与精度；最终可升级到 YOLOv8m。

### 4.2 训练超参数（参考项目参数 + 交通标志适配）

```yaml
# 训练配置
task: detect
mode: train

# 模型
model: yolov8s.pt          # 使用预训练权重微调

# 数据
data: datasets/dataset/data.yaml

# 训练参数
epochs: 300                 # 与参考项目一致
batch: 16                   # 根据 GPU 显存调整（参考项目用 8）
imgsz: 640                  # 输入图像尺寸
optimizer: SGD              # 与参考项目一致
lr0: 0.01                   # 初始学习率
lrf: 0.01                   # 最终学习率（余弦退火）
momentum: 0.937
weight_decay: 0.0005
warmup_epochs: 3.0
warmup_momentum: 0.8
warmup_bias_lr: 0.1

# 数据增强
mosaic: 1.0                 # Mosaic 增强
mixup: 0.0                  # MixUp（可尝试开启 0.1）
fliplr: 0.5                 # 水平翻转
scale: 0.5                  # 缩放
translate: 0.1              # 平移
hsv_h: 0.015                # 色调增强
hsv_s: 0.7                  # 饱和度增强
hsv_v: 0.4                  # 亮度增强
degrees: 10.0               # 旋转（交通标志可能倾斜）

# 其他
amp: true                   # 混合精度训练
device: 0                   # GPU 编号
workers: 8                  # 数据加载线程数
patience: 50                # 早停耐心值
pretrained: true
exist_ok: false
```

### 4.3 训练流程

```text
步骤 1: 数据准备
  ├── 下载 CCTSDB 数据集（主数据集）
  ├── 下载 TT100K 数据集（补充数据集，可选）
  ├── 编写类别映射脚本（统一两个数据集的类别体系）
  ├── 使用 LabelImg 标注（若需补充标注）
  ├── 运行 xml2txt.py 将 VOC XML 转为 YOLO TXT
  ├── 运行 split_data.py 划分 train/val/test
  └── 配置 data.yaml

步骤 2: 模型训练
  ├── 选择 YOLOv8s 预训练权重
  ├── 配置训练超参数
  ├── 启动训练（300 epochs）
  ├── 监控训练曲线（loss / mAP / precision / recall）
  └── 保存 best.pt 和 last.pt

步骤 3: 模型评估
  ├── 在 test 集上评估 mAP50 / mAP50-95
  ├── 生成混淆矩阵
  ├── 分析各类别的 PR 曲线
  └── 错误案例分析

步骤 4: 模型导出
  ├── 导出 ONNX 格式（可选，用于跨平台部署）
  └── 导出 TorchScript 格式（可选）
```

### 4.4 训练命令

```python
from ultralytics import YOLO

# 加载预训练模型
model = YOLO('yolov8s.pt')

# 开始训练
results = model.train(
    data='datasets/dataset/data.yaml',
    epochs=300,
    batch=16,
    imgsz=640,
    optimizer='SGD',
    lr0=0.01,
    device=0,
    patience=50,
    amp=True,
    project='runs/detect',
    name='traffic_sign_v1'
)
```

### 10.2 交通标志识别的特殊考虑

| 考虑点 | 说明 |
|--------|------|
| 小目标检测 | 交通标志在图片中可能较小，建议使用较高分辨率（640 或 1280） |
| 类别不平衡 | 限速标志数量远多于其他类别，训练时注意数据增强平衡 |
| 形状多样性 | 圆形、三角形、矩形等多种形状，YOLOv8 的 anchor-free 机制天然适配 |
| 颜色特征 | 红色（禁令）、蓝色（指示）、黄色（警告）是重要特征，HSV 增强需保留 |
| 遮挡/模糊 | 实际场景中标志可能被遮挡，需在数据增强中模拟 |
| 夜间/雨天 | 考虑不同光照条件下的鲁棒性 |
