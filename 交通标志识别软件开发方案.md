## 三、数据集方案（TT100K）

### 3.1 数据集介绍

#### TT100K（清华-腾讯交通标志数据集）

| 项目 | 说明 |
|------|------|
| 全称 | Tsinghua-Tencent 100K |
| 来源 | 清华大学与腾讯联合发布 |
| 数据量 | 100,000 张街景图片，30,000+ 个标注实例 |
| 类别数 | 221 类 |
| 标注格式 | JSON 格式，含边界框 `[x, y, w, h]` 和类别信息 |
| 下载地址 | [TT100K 官方页面](https://cg.cs.tsinghua.edu.cn/traffic-sign/) |
| Ultralytics 文档 | [TT100K 数据集文档](https://docs.ultralytics.com/zh/datasets/detect/tt100k/) |

**选择理由**：
- 数据量大（100K 张街景图片），场景多样性好，来自腾讯街景数据
- 涵盖中国所有常见交通标志类别
- Ultralytics 官方支持，有标准的 YAML 配置
- 适合高精度、多类别交通标志识别

### 3.2 数据集目录结构

下载解压后，TT100K 目录应包含：

```

│       └── annotations.json     # 测试集标注
├── data.yaml                    # YOLO 数据集配置（训练时自动下载数据集）
├── TT100K/                      # 数据集（训练时自动生成）

```

### 3.3 标签格式

标准 YOLO TXT 格式：

```
<class_id> <x_center> <y_center> <width> <height>
```

所有坐标归一化到 [0, 1]。每张图片对应一个同名 `.txt` 文件。

TT100K 原始标注为 JSON 格式，训练时由 Ultralytics 自动转换为 YOLO TXT 格式。

### 3.4 交通标志类别设计（TT100K 221 类）

使用 TT100K 完整的 221 类，与 Ultralytics 标准一致。按前缀分组：

| 前缀 | 含义 | 示例 | 数量 |
| ---- | ---- | ---- | ---- |
| `pl*` | 限速标志 | pl5, pl20, pl60, pl120 | ~30 |
| `pm*` | 限重标志 | pm10, pm30, pm55 | ~15 |
| `p*` | 禁令/指示标志 | pn, pne, ps, p1~p28 | ~90 |
| `w*` | 警告标志 | w1, w8, w13, w22 | ~45 |
| `i*` | 信息标志 | i1~i15, io, ip | ~20 |
| `il*` | 限速信息标志 | il60, il80, il100 | ~4 |
| `pa*`, `pb*`, `pc*`, `pg*`, `ph*`, `pi*`, `po*`, `pr*` | 其他标志 | pa8, pr30, ph4 | ~17 |

完整类别列表见 `data.yaml` 或 [Ultralytics TT100K 文档](https://docs.ultralytics.com/zh/datasets/detect/tt100k/)。

## 四、模型训练方案

### 4.1 模型选择

与参考项目一致，使用 **YOLOv8** 目标检测模型：

| 模型变体 | 参数量 | mAP (COCO) | 适用场景 | 推荐 |
|----------|--------|------------|----------|------|
| YOLOv8n | 3.2M | 37.3 | 轻量部署、实时性要求高 | 入门调试 |
| **YOLOv8s** | **11.2M** | **44.9** | **平衡精度与速度** | **推荐** |
| YOLOv8m | 25.9M | 50.2 | 更高精度 | 有 GPU 资源时 |
| YOLOv8l | 43.7M | 52.9 | 高精度 | 大规模部署 |
| YOLOv8x | 68.2M | 53.9 | 最高精度 | 追求极致精度 |

**建议**：开发阶段使用 **YOLOv8s**，兼顾训练速度与精度；最终可升级到 YOLOv8m。

### 4.2 训练超参数（参考项目参数 + 交通标志适配）

```yaml
# 训练配置
task: detect
mode: train

# 模型
model: yolov8s.pt          # 使用预训练权重微调

# 数据
data: datasets/dataset/data.yaml

# 训练参数
epochs: 300                 # 与参考项目一致
batch: 16                   # 根据 GPU 显存调整（参考项目用 8）
imgsz: 640                  # 输入图像尺寸
optimizer: SGD              # 与参考项目一致
lr0: 0.01                   # 初始学习率
lrf: 0.01                   # 最终学习率（余弦退火）
momentum: 0.937
weight_decay: 0.0005
warmup_epochs: 3.0
warmup_momentum: 0.8
warmup_bias_lr: 0.1

# 数据增强
mosaic: 1.0                 # Mosaic 增强
mixup: 0.0                  # MixUp（可尝试开启 0.1）
fliplr: 0.5                 # 水平翻转
scale: 0.5                  # 缩放
translate: 0.1              # 平移
hsv_h: 0.015                # 色调增强
hsv_s: 0.7                  # 饱和度增强
hsv_v: 0.4                  # 亮度增强
degrees: 10.0               # 旋转（交通标志可能倾斜）

# 其他
amp: true                   # 混合精度训练
device: 0                   # GPU 编号
workers: 8                  # 数据加载线程数
patience: 50                # 早停耐心值
pretrained: true
exist_ok: false
```

### 4.3 训练流程

```text
步骤 1: 数据准备
  ├── 配置 data.yaml（Ultralytics 训练时自动下载 TT100K）
  └── 无需手动下载和转换

步骤 2: 模型训练
  ├── 选择 YOLOv8s 预训练权重
  ├── 配置训练超参数
  ├── 启动训练（300 epochs）
  ├── 监控训练曲线（loss / mAP / precision / recall）
  └── 保存 best.pt 和 last.pt

步骤 3: 模型评估
  ├── 在 test 集上评估 mAP50 / mAP50-95
  ├── 生成混淆矩阵
  ├── 分析各类别的 PR 曲线
  └── 错误案例分析

步骤 4: 模型导出
  ├── 导出 ONNX 格式（可选，用于跨平台部署）
  └── 导出 TorchScript 格式（可选）
```

### 4.4 训练命令

```python
from ultralytics import YOLO

# 加载预训练模型
model = YOLO('yolov8s.pt')

# 开始训练
results = model.train(
    data='datasets/dataset/data.yaml',
    epochs=300,
    batch=16,
    imgsz=640,
    optimizer='SGD',
    lr0=0.01,
    device=0,
    patience=50,
    amp=True,
    project='runs/detect',
    name='traffic_sign_v1'
)
```

## 七、核心代码模块设计

### 7.1 `Config.py`

配置文件，包含 221 类交通标志的中英文名称映射和类别分组。详见 `Config.py`。

核心工具函数：
- `drawRectBox()` — 绘制边界框 + 中文标签
- `img_cvread()` — 支持中文路径的图片读取
- `cvimg_to_qpiximg()` — OpenCV → QPixmap 转换
- `cv2AddChineseText()` — 中文文字渲染

### 7.3 `MainProgram.py`（与参考项目一致的架构）

核心类：
- `MainWindow(QMainWindow)` — 主窗口，包含所有检测逻辑
- `btn2Thread(QThread)` — 视频保存后台线程

## 十、关键注意事项

### 10.1 与参考项目保持一致的部分

### 10.2 交通标志识别的特殊考虑

| 考虑点 | 说明 |
|--------|------|
| 小目标检测 | 交通标志在图片中可能较小，建议使用较高分辨率（640 或 1280） |
| 类别不平衡 | 限速标志数量远多于其他类别，训练时注意数据增强平衡 |
| 形状多样性 | 圆形、三角形、矩形等多种形状，YOLOv8 的 anchor-free 机制天然适配 |
| 颜色特征 | 红色（禁令）、蓝色（指示）、黄色（警告）是重要特征，HSV 增强需保留 |
| 遮挡/模糊 | 实际场景中标志可能被遮挡，需在数据增强中模拟 |
| 夜间/雨天 | 考虑不同光照条件下的鲁棒性 |

### 10.3 可选增强功能（非必须，按需开发）

- **模型导出**：ONNX / TensorRT 部署到嵌入式设备
- **历史记录**：将检测结果存入 SQLite 数据库
- **统计报表**：生成检测统计报告（PDF/Excel）
- **Web 版本**：使用 Flask/FastAPI 提供 REST API
